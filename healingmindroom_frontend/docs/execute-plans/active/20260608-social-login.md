# Execute Plan: 소셜 로그인 (Kakao & Google) 추가 구현

본 계획서는 Healing Mind Room MVP 내에 카카오 및 구글 소셜 로그인 인터페이스를 연동하고 상태를 제어하는 단계별 실행 명세입니다.

## 1. 개발 마일스톤 및 완료 조건
- **목표**: 서비스 최전방에 로그인 카드(Step 0)를 배치하여, 카카오/구글 로그인이 성공한 유저에 한해서만 치유의 방 메인 고민 작성 홈으로 진입할 수 있도록 구현합니다.
- **기간**: 2026.06.08

## 2. 상세 작업 태스크

### 2.1 문서 명세 갱신
- [x] PRD, ARCHITECTURE, DATA_MODEL, UI_UX_FLOW, SECURITY_NOTES 보완 업데이트.
- [x] `.env.example` 에 `VITE_ENABLE_REAL_SOCIAL=false` 설정 및 소셜 클라이언트 ID 템플릿 기재.

### 2.2 타입 및 컨텍스트 스토어 확장
- [x] `src/types/index.ts` 에 `User` 타입(name, email, avatarUrl, provider) 추가.
- [x] `src/context/FlowContext.tsx` 에 `user` 상태, `login(provider)` 및 `logout()` 비즈니스 메소드 구현.
- [x] 로컬스토리지(`healing_session`)로부터 초기 로그인 상태를 복구하는 `useEffect` 연동.

### 2.3 UI 컴포넌트 개발
- [x] `src/components/Step0Login.tsx` 신규 컴포넌트 마크업.
  - 카카오 로그인 스타일 가이드 준수 (노란색 #FEE500, 로고 이미지/이모지 💬)
  - 구글 로그인 스타일 가이드 준수 (흰색 #FFFFFF, 얇은 보더, 로고 이미지/이모지 🌐)
  - 30~60대 대상 큰 글씨 및 클릭하기 편한 56px 높이 버튼 적용.
- [x] `src/App.tsx` 의 헤더 영역 업데이트.
  - 로그인 상태인 경우 우측 상단에 둥근 프로필 아바타, 닉네임, 그리고 로그아웃 버튼 노출.
  - 라우팅 가드: `user === null` 이면 무조건 `Step0Login` 렌더링.

### 2.4 Mock 로그인 시뮬레이션 구현
- [x] 실제 클라이언트 ID가 미설정 상태인 경우, 클릭 시 1초 동안 "안전하게 연동 중입니다..." 문구와 함께 로딩 마스크가 뜬 후, 힐링 가상의 프로필 정보(예: "홍길동", "gildong@example.com")를 생성하여 로그인 완료 세션을 바인딩.

## 3. 검증 계획
- **로그인 차단 검증**: 로그인하지 않은 상태에서 브라우저 기동 시 홈 화면이 노출되지 않고 로그인 카드가 강제 표출되는지 검증.
- **Mock 시뮬레이션 검증**: 카카오/구글 로그인 버튼 클릭 시 Mock 유저 프로필이 정상 바인딩되어 헤더에 연동되고 홈 화면으로 자연스럽게 진입하는지 확인.
- **세션 유지 검증**: 로그인 성공 후 새로고침(F5)을 해도 로그인 상태가 풀리지 않고 유지되는지 로컬 스토리지 데이터 검사.
- **로그아웃 검증**: 로그아웃 버튼 클릭 시 세션 정보가 소멸되고 다시 로그인 카드로 리디렉션되는지 확인.
